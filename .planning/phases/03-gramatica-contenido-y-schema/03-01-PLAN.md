<objective>
Make the frozen content contract able to carry rich grammar-exercise data, additively.
Two capabilities: (1) grammar example sentences may carry word-level tokens[] (already
schema-allowed) so Phase 4 can build reorder-the-sentence chips; (2) a new optional
per-token `blank` flag marks which token(s) are the answer, so Phase 4 can build
fill-in-the-particle/form cloze. No app feature code here — only schema, validator,
structuring rules, and regenerated types.
</objective>

  `additionalProperties:false` — the new property must be declared so tokens carrying it
  validate. **Verify:** the 5 existing classes (which have no `blank`) still validate.
- `$defs.sentence.tokens` already exists — confirm no schema change needed for reorder.
- **done:** `node skill/validate.mjs content/classes/2026-07-13.json` (and the other 4)
  still prints `VALID`.

### Task 2 — Extend validators to grammar example tokens
- In `skill/validate.mjs`: today the 3 renderability invariants run on vocab headword
  tokens (and optionally vocab example tokens). Extend them to run on ANY grammar
  `examples[].tokens` when present: concat(surface)===kanji, concat(reading)===kana,
  every isKanji token has non-empty kanji[]. `blank` needs no extra rule (ajv checks the
  boolean type). Keep it fail-closed (exit 1 with reason).
- Mirror the same change into `app/src/content/validate.mjs` (keep the two in lockstep;
  diff them to confirm parity).
- **verify:** craft a temp class with a valid grammar example (tokens+blank) → VALID;
  mutate one surface so concat != kanji → exit 1 with a clear message.

### Task 3 — Update the structuring prompt (structure.md)
- In `skill/prompts/structure.md`, in the grammar section (§5), add rules:
  - Emit word/bunsetsu-level `tokens[]` on each grammar example (chips the learner will
    reorder): split into meaningful units (これ / は / ほん / です), same renderability
    invariants as vocab; kana-only chips are `isKanji:false` with surface===reading.
  - Mark the token(s) the pattern TEACHES with `"blank": true` (the particle は/を/の/が,
    or the verb form) — that becomes the cloze answer. Usually exactly one.
  - Keep it OPTIONAL-friendly: if word-splitting a sentence is uncertain, a single
    whole-sentence token is still valid; never emit wrong readings.
- Add a short worked example (e.g. これはほんです with は marked blank).
