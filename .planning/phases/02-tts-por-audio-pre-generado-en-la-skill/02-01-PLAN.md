Implementation (in order):
1. Read + validate FIRST: `import { validateClass } from './validate.mjs'` — parse argv[2],
   JSON.parse, validateClass(doc); on any error print `REFUSED (validation failed): <msg>` and
   exit 1 (exact commit-class.mjs step-1 pattern).
2. Preflight guards (fail-closed):
   - `execFileSync('say', ['-v', '?'])` output must contain 'Kyoko'; else exit 1 with message
     `REFUSED: Kyoko voice not installed (System Settings -> Accessibility -> Spoken Content)`.
     Never generate with a non-Japanese voice (T-02-05, mirrors TTS-02).
   - `doc.classId` must match `/^\d{4}-\d{2}-\d{2}(-\d+)?$/`; else exit 1 (T-02-02).
3. Build the entry list from doc.vocab ONLY (D-04 — grammar/kanji/notes get NO audio, v2):
   for each item: `[item.id, item.kana]`, plus `[item.id + ':example', item.example.kana]`
   when `item.example?.kana` exists.
4. For each (key, kana): filename = `key.replaceAll(':', '_') + '.m4a'`; assert it matches
   `/^[A-Za-z0-9_-]+\.m4a$/` else exit 1 (path-traversal guard T-02-02). relPath =
   `audio/${doc.classId}/${filename}` (relative to content/). kanaHash = `'sha256-' +
   createHash('sha256').update(kana, 'utf8').digest('hex')`.
5. Load manifest `content/audio/index.json` if it exists, else default
   `{ audioVersion: 1, generatedAt: '', files: {}, kanaHashes: {} }` (shape in <interfaces>).
6. SKIP an entry when the target file exists AND `manifest.kanaHashes[key] === kanaHash`.
   Otherwise generate:
   - `mkdtempSync(join(tmpdir(), 'jp-audio-'))`; write kana to `<tmp>/text.txt` (utf8).
   - `execFileSync('say', ['-v', 'Kyoko', '-o', '<tmp>/out.aiff', '-f', '<tmp>/text.txt'])`
     — text goes through a FILE, never argv/shell (T-02-01; same execFileSync-array pattern
     as commit-class.mjs:80). No rate flag (default Kyoko rate; speed controls are out of scope).
   - `execFileSync('ffmpeg', ['-y', '-i', '<tmp>/out.aiff', '-ac', '1', '-c:a', 'aac',
     '-b:a', '48k', '<abs content/audio/classId/filename>'], { stdio: ['ignore', 'ignore', 'inherit'] })`
     — ffmpeg chosen over afconvert per D-01 option B: already a documented skill dependency
     (transcribe.sh), verified installed (8.1). mkdirSync the class dir recursive first.
   - rmSync the tmp dir `{ recursive: true, force: true }` in a finally block (T-02-04).
   - Set `manifest.files[key] = relPath; manifest.kanaHashes[key] = kanaHash`; count generated++.
7. Orphan cleanup (D-09): in `content/audio/<classId>/`, unlinkSync every `.m4a` not in the
   expected filename set (count deleted++). Remove every manifest.files/kanaHashes key that
   starts with `${doc.classId}:` but is not in the current key set.
8. Idempotent manifest write: rebuild `files` and `kanaHashes` with keys sorted alphabetically
   (deterministic bytes). Compare `JSON.stringify({files, kanaHashes})` before/after; ONLY on
   real change set `generatedAt = new Date().toISOString().replace(/\.\d{3}Z$/, 'Z')` and write
   `JSON.stringify(manifest, null, 2) + '\n'` (commit-class.mjs:62-70 pattern).
9. Copy to public (commit-class.mjs:72-77 pattern): mkdirSync `app/public/content/audio`
   recursive; `cpSync(content/audio/<classId>, app/public/content/audio/<classId>, { recursive: true })`
   when the dir exists; copyFileSync the manifest to `app/public/content/audio/index.json`.
   (Root .gitignore already ignores app/public/content/ entirely — verified in 02-PATTERNS.md.)
10. Final stdout line (exact contract): `AUDIO <classId> generated=<g> skipped=<s> deleted=<d>`.
