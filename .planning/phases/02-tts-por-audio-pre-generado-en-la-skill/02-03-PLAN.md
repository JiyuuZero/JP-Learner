<objective>
Delivery hardening for pre-generated audio: the build pipeline copies content/audio/ from the
canonical source (D-08), and the service worker caches .m4a binaries CacheFirst while the JSON
manifest stays NetworkFirst (D-06) — without inflating the SW precache. Ends with the human
checkpoint that closes the phase: hearing Kyoko audio play in the browser.

Purpose: without the sync-content change, `npm run build` DELETES the audio copies (it rm-rf's
app/public/content/ and recopies) — audio would vanish from every deploy. Without the cache rule
order fix, the existing generic /content/ NetworkFirst rule swallows audio requests.
Output: modified app/scripts/sync-content.mjs + app/vite.config.ts, verified build, user-approved
playback.

  ```ts
    options: {
      cacheName: 'content-audio',
      expiration: { maxEntries: 300, maxAgeSeconds: 60 * 60 * 24 * 30 }, // T-02-11: bounded staleness
    },
  },
  ```
