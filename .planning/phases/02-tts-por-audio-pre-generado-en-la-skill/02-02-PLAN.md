```ts
export type AudioManifest = Record<string, string>
export function exampleKey(itemId: string): string           // `${itemId}:example`
export function parseAudioManifest(raw: unknown): AudioManifest
export async function loadAudioManifest(): Promise<AudioManifest>
export function playAudio(manifest: AudioManifest, key: string): boolean
export function stopAudio(): void
```

- `parseAudioManifest` (PURE — unit-tested): raw must be a non-array object whose `files` is a
  non-array object; keep only entries whose value is a string matching
  `/^audio\/[A-Za-z0-9_\-/]+\.m4a$/` (T-02-07 allowlist — rejects `..`, absolute/external URLs,
  wrong extensions); anything else -> `{}` / dropped.
- `loadAudioManifest`: `const res = await fetch(`${BASE}content/audio/index.json`)` with
  `const BASE = import.meta.env.BASE_URL` (store.ts pattern). If `!res.ok` return `{}` (D-03:
  404 is NOT an error — divergence from store.ts fetchJson). Wrap `await res.json()` in
  try/catch returning `{}` — GitHub Pages SPA fallback serves HTML with HTTP 200 (T-02-06).
- `playAudio`: module-level `let current: HTMLAudioElement | null = null`. Look up
  `manifest[key]`; if absent return false. If `typeof speechSynthesis !== 'undefined'` call
  `speechSynthesis.cancel()` (never overlap voices). If `current` exists: `current.pause();
  current.currentTime = 0` (the speechSynthesis.cancel() analog for audio). Then
  `current = new Audio(`${BASE}content/${path}`)` and `void current.play().catch(() => undefined)`
  — a rejected play() (uncached offline file, autoplay policy) degrades SILENTLY, never falls
  through to a wrong voice. Return true. Comment: MUST be called inside a tap handler
  (iOS user-gesture requirement, TTS-01).
- `stopAudio`: pause + rewind `current` if set.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network -> app | audio manifest + .m4a fetched from GitHub Pages (SPA fallback can serve HTML with 200) |
| manifest data -> URL construction | manifest values become Audio src URLs |
| user tap -> playback | audio/speech must only ever start from a user gesture |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Spoofing | loadAudioManifest (Pages serves HTML as 200) | mitigate | `!res.ok -> {}` plus try/catch around `.json()` plus parseAudioManifest shape validation — any failure yields an empty map, app boots normally |
| T-02-07 | Tampering | manifest values -> `new Audio(url)` | mitigate | per-entry allowlist regex `/^audio\/[A-Za-z0-9_\-/]+\.m4a$/` in parseAudioManifest; invalid entries dropped (blocks `..`, absolute URLs, non-m4a) |
| T-02-08 | Elevation of user intent (auto-play) | playAudio / speak | mitigate | playback only inside SpeakerButton's tap handler; no play on mount/load; rejected play() degrades silently |
| T-02-09 | Spoofing (wrong voice) | fallback chain tail | mitigate | chain ends in speakJa which is a silent no-op without a ja-JP voice; button hidden when neither capability exists (TTS-02 verbatim) |
| T-02-10 | Information disclosure | static class audio served publicly | accept | content is already public in the repo/Pages; personal study vocabulary, no PII |
</threat_model>
