<objective>
Build the Claude Code content skill: the producer half of the system that never deploys to Pages. It transcribes class audio locally with whisper.cpp large-v3 (no API key), structures the mixed ES/JA transcript into schema-valid JSON (Claude reasoning is the disambiguator), validates against the FROZEN `content.schema.json` with ajv + the 3 cross-field invariants before writing, and writes/commits content with deterministic IDs + class/date tags — always under one fixed documented contract.

Purpose: This is the only supported way to add classes. It depends ONLY on the frozen schema (Plan 01), so it runs in parallel with the app shell (Plan 02). Enforcing the contract at the producer boundary guarantees the app never receives invalid content.
Output: `skill/SKILL.md`, `skill/transcribe.sh`, `skill/validate.mjs`, `skill/prompts/structure.md`, `skill/commit-class.mjs`, `skill/package.json`, and a `.gitignore` entry for `audio-src/`.
</objective>

<!-- From Plan 01 (frozen). The skill validates against the SAME schema file; it never redefines the shape. -->
Frozen schema at: content/schema/content.schema.json
Class file target: content/classes/<classId>.json  (classId = YYYY-MM-DD or YYYY-MM-DD-<n>)
Manifest: content/index.json  { contentVersion, generatedAt, classes:[{ id,date,label,file,contentHash,counts }] }

ID rule (validate.mjs enforces): every item id === `<classId>:<type>:<slug>`, type in vocab|grammar|kanji|note.
  slug: vocab = hepburn-ascii of the kana reading (lowercased, non-[a-z0-9] to '-', collapsed, trimmed);
        grammar = pattern slug; kanji = the literal kanji char; note = ordinal. Collision -> append -2,-3 by document order.
3 cross-field invariants (imperative, from Plan 01's validate.mjs): surface-concat===kanji, reading-concat===kana, isKanji token has non-empty kanji[].

whisper-cli flags (verified, 01-RESEARCH.md The Skill): -m <model> -f <wav> -l auto -oj -of <prefix> ; NEVER --translate.
Model: ggml-large-v3 (turbo only if memory-constrained). ffmpeg -> 16kHz mono 16-bit WAV (-ar 16000 -ac 1 -c:a pcm_s16le).
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| class audio -> transcript | Local audio processed on the Mac; never leaves the machine (no keys, no cloud). |
| transcript -> structured JSON | Claude reasoning step produces candidate content; untrusted until validated. |
| candidate JSON -> validate.mjs | The producer-side contract gate: schema + invariants + ID determinism, fail-closed before any write/commit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Invalid/hallucinated content committed (Whisper silence-hallucination, wrong readings) | mitigate | validate.mjs (ajv + 3 cross-field invariants + ID determinism) refuses to write on any failure; human-in-the-loop verification documented in SKILL.md (the user attended the class). |
| T-03-02 | Tampering | Non-deterministic IDs orphaning SRS history on re-process (PROG-04) | mitigate | Deterministic slug rule enforced by validate.mjs; commit-class.mjs upserts (never duplicates) so re-processing is idempotent. |
| T-03-03 | Information disclosure | API key leaking via the skill / audio uploaded to a cloud service | mitigate | whisper.cpp runs fully local with no API key (hard constraint); `--translate` never invoked; audio-src is gitignored so raw audio is never committed. |
| T-03-04 | Repudiation | Cannot tell which content came from which class/date | accept | Deterministic classId-prefixed IDs + class/date tags + git commit history provide full provenance. |
</threat_model>
