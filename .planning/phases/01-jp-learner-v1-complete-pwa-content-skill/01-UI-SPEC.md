# Phase 1 — UI Design Contract

> Visual and interaction contract for JP-Learner v1. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Design source of truth: `references/reference_0..3.png` (as references, not pixel-perfect clones) + CONTEXT.md decisions D-01..D-05.
> **Interface language is SPANISH.** Japanese *content* renders via 3 display modes (A/B/C) using native `<ruby>`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (no component library) |
| Preset | not applicable |
| Component library | **none** — hand-rolled React 19 components + Tailwind 4 (CSS-first `@theme`) |
| Icon library | **lucide-react** (tree-shakeable, matches the thin line-icon look of ref_0/ref_2 bottom nav) |
| Font | **Nunito** (headings + UI — rounded, bold sans-serif matching ref_1/ref_2 headings). Japanese: system CJK stack `"Hiragino Sans", "Noto Sans JP", sans-serif`. Romaji furigana in the same UI font. |

**Why no component library (locked upstream, not re-asked):** STACK.md and SUMMARY.md explicitly rule out shadcn/MUI/Radix and any JA-processing libs — the app is a static, offline, token-budgeted single-phase build for a lighter LLM. Components are hand-rolled over Tailwind 4 utilities. `shadcn init` is therefore intentionally NOT run; the shadcn/registry gate is not applicable and no third-party registries are in play.

**Global visual language (applies to every component):**
- Heavily rounded corners (cards `rounded-3xl` / 24px, buttons full `pill`, chips `rounded-2xl`).
- Soft, low-contrast shadows (`shadow-soft`, defined below) — never hard black drop shadows.
- Bold rounded headings, generous whitespace, one primary action per screen.
- Mobile-first: design at 375–430px width; content column max ~480px, centered on desktop.
- Fixed bottom navigation on all top-level screens; exercise screens replace it with a bottom action button.

---

## Spacing Scale

Declared values (multiples of 4; maps to Tailwind 4 default spacing so `p-2`=8, `p-4`=16, etc.):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, ruby `<rt>`-to-base gap, inline padding |
| sm | 8px | Chip inner padding, compact stacks, points-pill padding |
| md | 16px | Default gap between elements, card inner padding (small cards), screen horizontal padding |
| lg | 24px | Card inner padding (large cards), section vertical rhythm, card corner radius |
| xl | 32px | Gap between major sections on the dashboard |
| 2xl | 48px | Top-of-screen header offset, empty-state vertical centering |
| 3xl | 64px | Bottom-nav clearance / safe-area bottom padding on scrollable screens |

**Exceptions:**
- Touch targets: any tappable control (icon buttons, chips, nav items, grade buttons) has a **minimum hit area of 44×44px** even if the visual glyph is smaller.
- Bottom-nav bar height: **64px** plus the device `env(safe-area-inset-bottom)`.
- Full-width primary action button height: **56px**.

## Typography

Font family: **Nunito** for all Spanish UI text; CJK system stack for Japanese base glyphs. Exactly 4 sizes, exactly 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 800 (extrabold) | 1.2 | Onboarding headline, big screen titles ("Familia" term on the exercise card renders larger — see note) |
| Heading | 20px | 700 (bold) | 1.25 | Section titles ("Categorías", "Reto de hoy", screen titles in the top bar), card titles |
| Body | 16px | 400 (regular) | 1.5 | All body copy, translations, example sentences, button labels (button label 16px/700) |
| Label | 14px | 400 (regular) | 1.4 | Secondary/caption text ("En japonés", streak sublabel, progress counters, chip text, empty-state body) |

**Weights: only 2 — regular (400) and bold (700).** Display uses 800 as the single heavier accent for the onboarding headline only; do not introduce a 4th weight elsewhere. Bold (700) is the "semibold/bold" for headings and CTA labels.

**Japanese content type sizing (per display mode):**
- Exercise-card headline term (the studied word, e.g. ref_3 "Familia"): render at **28px display**, but the JA base glyph may scale up to 32–36px for legibility of kanji — this is a content-legibility exception, not a new UI size token.
- Furigana `<rt>`: **~55–60%** of its base font size (native `<ruby>` default is acceptable). Romaji furigana (Mode A) uses the UI font; kana furigana (Mode B) uses the CJK stack.

## Color

Palette locked from the reference images (sampled hex). 60 / 30 / 10 split. This is a **playful pastel** system, so pastel tints are treated as *surface variants inside the 30% band*, not as accent — accent (10%) is strictly the indigo/coral/yellow signal colors listed below.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FCFCFC` (near-white) on `#F5F5F8` app background | Page background, scroll surface, most negative space |
| Secondary (30%) | White `#FFFFFF` cards + pastel card tints: lavender `#E7E7FC`, mint `#CFF0E4`, peach `#FCF0E7`, soft-blue `#C6DAF2`; dark card `#30303C` | Cards, category tiles, glossary items, bottom-nav pill (dark), "continue" card (dark) |
| Accent (10%) | Indigo `#5A5AE6` (primary) | See reserved-for list below |
| Accent-2 (10%, sparing) | Golden yellow `#EEC24E` | Points/gems pill, streak flame, "learned" star/marker, celebration accents |
| Coral (accent, sparing) | `#E46C60` | Active bottom-nav item highlight (ref_0 home square), the single "start/primary tab" highlight |
| Destructive | `#DC2626` | Import-overwrite confirmation, "borrar/reiniciar progreso" only |

**Accent (indigo `#5A5AE6`) reserved for — explicit list (never "all interactive elements"):**
1. The single primary/full-width action button per screen ("Empezar a aprender", "Seguir", "Comprobar", "Continuar").
2. The active/filled state of the review-scope segmented control (Hoy / Semana / Total).
3. Progress-bar fill and the numbered-step active indicator on the exercise screen.
4. Correct-answer / selected word-bank chip fill.
5. Text links and the active glossary-view toggle.

**Pastel tints (30% band) reserved for — explicit list:**
- Category / class tiles on the dashboard (rotate lavender → mint → peach → soft-blue for visual variety, deterministically by index).
- The large lavender term card on the exercise screen (`#E7E7FC`).
- Word-bank chips: neutral pastel by default; indigo fill only when placed correctly.

**Dark surface (`#30303C`) reserved for:** the "Continuar donde lo dejaste" card and the bottom-nav pill (per ref_2). Do not use dark surfaces elsewhere.

**Golden yellow (`#EEC24E`) reserved for:** points counter pill, streak indicator, "kanji aprendido" star/toggle. Never as a button fill or large background.

Text colors: primary text `#1A1A24` (near-black), secondary/caption `#6B6B7B`, on-dark-surface `#FFFFFF` / `#C8C8D4`. Contrast target: all body/label text ≥ 4.5:1 on its surface; on indigo/dark surfaces use white.

## Copywriting Contract

Tone: **Spanish, warm, encouraging, non-punitive, informal "tú".** No streaks-lost guilt, no "fallaste". Celebrate progress; frame errors as "casi" / "vuelve a intentarlo".

| Element | Copy |
|---------|------|
| Onboarding CTA | **Empezar a aprender** (with → icon, ref_1) |
| Primary CTA (exercise, mid-session) | **Seguir** |
| Primary CTA (exercise, needs check first) | **Comprobar** → after check → **Seguir** |
| Primary CTA (dashboard continue) | **Continuar** (on the dark "Continuar donde lo dejaste" card) |
| Session finish CTA | **Terminar sesión** |
| Empty state heading (no content loaded yet) | **Aún no hay clases** |
| Empty state body (no content) | **Añade una clase con la skill para empezar a practicar.** (with a passive hint; no in-app action since content comes from the skill/git) |
| Empty state — nothing due today | Heading: **¡Todo al día!** · Body: **No tienes repasos pendientes hoy. Puedes repasar un periodo entero cuando quieras.** |
| Empty state — glossary search no results | Heading: **Sin resultados** · Body: **No encontramos nada para «{query}». Prueba con otra palabra.** |
| Loading state | **Cargando tu contenido…** (skeleton cards, not a bare spinner) |
| Error state (content fetch failed) | Heading: **No se pudo cargar el contenido** · Body: **Revisa tu conexión y vuelve a intentarlo.** + **Reintentar** button |
| No-TTS-voice state | The speaker button is **hidden** (not disabled) when no `ja-JP` voice exists. If a user somehow triggers TTS with no voice: silent no-op — **never** speak with a non-Japanese voice, never show a blocking error. Optional inline label: **Pronunciación no disponible en este dispositivo.** |
| Answer feedback — correct | **¡Bien!** (green tick, indigo/mint accent) |
| Answer feedback — incorrect | **Casi. La respuesta es: {answer}** (never "Incorrecto"/"Mal") |
| Self-grade buttons (flashcard) | **Otra vez** · **Difícil** · **Bien** · **Fácil** (maps to FSRS Again/Hard/Good/Easy) |
| Review-scope tabs | **Hoy** · **Semana** · **Total** |
| Review sub-mode selector (D-02) | **Repaso SRS** (lo que toca) · **Repasar periodo** (todo el bloque) |
| Display-mode selector (settings, DISP-01) | **A · Romaji sobre kana/kanji** · **B · Kanji progresivo** · **C · Solo kanji** |
| Export CTA | **Exportar progreso** |
| Import CTA | **Importar progreso** |
| Destructive confirmation — Import overwrite | **Importar progreso**: "Esto reemplazará todo tu progreso actual por el del archivo. Esta acción no se puede deshacer. ¿Continuar?" · Buttons: **Cancelar** / **Reemplazar** (Reemplazar in destructive `#DC2626`) |
| Destructive confirmation — Reset (if built) | **Reiniciar progreso**: "Se borrará tu estado SRS, kanji aprendidos, racha y puntos. ¿Seguro?" · **Cancelar** / **Borrar todo** |

| Points label | **{n} pts** (with gem/star icon, non-punitive — points only ever go up) |
| Kanji-learned toggle | marked: **Aprendido** / unmarked: **Marcar como aprendido** |

## Screen & Component Contracts

Four top-level surfaces (bottom-nav shell) + the full-screen practice session. All mobile-first, single content column ≤480px.

### App shell / bottom navigation (UI-01, UI-02)
- Fixed bottom bar, 64px + safe-area. Four items: **Inicio · Glosario · Guardados · Perfil** with lucide icons (`home`, `book-open`, `bookmark`, `user`).
- Active item: filled indigo pill with white icon + visible label (ref_2 style). Inactive: thin line icon, `#6B6B7B`, no label.
- The bar is hidden during a practice session (replaced by the bottom action button) and hidden on onboarding.

### 1. Home dashboard (UI-03, GAM-01/02/03, SRS-03, D-01/D-02)
- Header row: greeting **"¡Hola!"** + avatar (right) + points pill (golden). Streak (flame + "{n} días") near the greeting.
- Global progress %: a circular or slim linear indicator (indigo fill) — mirrors ref_2's 69% ring.
- **Dark "Continuar donde lo dejaste" card** (`#30303C`, ref_2): shows current/last session context + circular % + **Continuar** button. Resumes the automatic hybrid session (D-01).
- **Practice-by-scope block:** segmented control **Hoy · Semana · Total** (indigo active), and beneath it the two sub-modes (D-02): **Repaso SRS** and **Repasar periodo**. Selecting scope + sub-mode launches the session.
- **"Reto de hoy" cards** (ref_2 mint/peach): quick-entry tiles, e.g. Vocabulario / Gramática, each a pastel rounded card with illustration/icon + count subtitle.

### 2. Practice session (EXER-01..06, SRS-02, D-01)
Shared session frame for all 5 exercise types:
- **Top:** back arrow (`←`, `ArrowLeft`) + centered title (exercise/category name) + points pill (right).
- **Progress indicator:** numbered step bar 1..N (ref_3) — filled indigo segment for current, gray for pending. Position at top, under the header.
- **Center:** the exercise body inside a large rounded card.
- **Bottom:** one full-width 56px pill action button (**Comprobar** → **Seguir**), pinned above safe-area. This replaces the bottom nav.

Exercise-type bodies:
- **Flashcard (EXER-01):** large lavender card `#E7E7FC` (ref_3) with the term (JA rendered per active display mode) + "En japonés/En español" sublabels + favorite heart (top-right) + a **Pronunciación** bar with speaker icon (TTS). Tap to reveal back (translation + example sentence + TTS). Then 4 self-grade buttons: **Otra vez / Difícil / Bien / Fácil**.
- **Multiple choice (EXER-02):** prompt card + 4 option chips (2-col grid); tap selects, correct fills indigo, wrong fills coral briefly; distractors from same-class vocab.
- **Fill-in / typing (EXER-03):** prompt card + text input (dashed underline like ref_3) + **Comprobar**. Tolerant checking (romaji/kana variants, n/nn). Feedback copy per contract.
- **Sentence word-bank (EXER-04) — the signature exercise, follow ref_3 closely:** JA sentence with gap(s) shown as a dashed slot; below it, a wrapped set of pastel word chips (`rounded-2xl`, alternating pastel tints exactly like ref_3). Tapping a chip moves it into the slot and fills it indigo; tapping again returns it. Include 1–2 distractor chips. **Comprobar** validates order.
- **ES↔JA matching (EXER-05):** two columns of cards (ES / JA); tap one then its pair; matched pairs animate out. Recognition-level, short sets.
- **Direction toggle (EXER-06):** each type practicable JA→ES and ES→JA; the prompt/answer sides swap. No separate screen — a small direction indicator in the header.

### 3. Library / Glossary (D-04, CONT-06, DISP-01..04)
- **Top:** title **"Glosario"** + global search field (rounded, `search` icon).
- **Two switchable views** (segmented toggle, indigo active): **Por clase** (list grouped by date → vocab/gramática/notas) and **Por categoría** (Vocabulario / Gramática).
- **List rows / cards:** each vocab item shows JA (per active display mode with `<ruby>`), romaji, ES translation, and a "Guardar/heart" toggle. Tapping opens item detail.
- **Item detail:** large card — kanji · kana · romaji · ES · example sentence (all per active display mode), TTS speaker, and a **kanji "Aprendido" toggle** per kanji (golden star) that feeds the learned-kanji set driving Mode B (DISP-03/04). Auto-marked kanji (D-03, FSRS-graduated) show the star pre-filled; user can toggle manually.

### 4. Settings / Perfil (DISP-01, PROG-02/03, TTS)
- **Display mode selector (DISP-01):** three-option control **A / B / C** with the labels above and a one-line Spanish explanation each.
- **Romaji visibility toggle** (show/hide romaji furigana in Mode A).
- **TTS toggle / status:** shows whether a `ja-JP` voice is available; hides speaker buttons app-wide if none (TTS-02).
- **Backup block:** **Exportar progreso** (downloads JSON) and **Importar progreso** (file picker → destructive-confirm → replace). Nag/hint copy encouraging periodic export (iOS eviction guardrail).

### Ruby rendering contract (DISP-01/02, all screens)
Japanese renders via native `<ruby>` from schema `tokens[]` — **never computed in-app**:
- **Mode A:** base = kanji/kana; `<rt>` = **romaji** over each reading segment. Romaji in UI font, `#6B6B7B`, ~58% size.
- **Mode B (progressive):** words default to **kana** (no ruby); a token whose kanji are all in the learned set renders as **kanji with kana furigana** in `<rt>`. Substitution is per-token, gradual (D-03).
- **Mode C:** kanji only, no `<rt>`. Provide `<rp>` fallback parentheses for non-ruby contexts.
- `<rt>` gap to base: 4px; never let furigana overlap adjacent lines (line-height 1.5 minimum on ruby-bearing text).

### Shared component tokens
- **Buttons:** primary = full-width 56px indigo pill, white 16px/700 label; secondary = white pill with indigo border/text; grade buttons = 4 equal pills (Otra vez neutral, Difícil peach, Bien mint, Fácil indigo-tint). Min hit area 44px.
- **Cards:** `rounded-3xl` (24px), `shadow-soft`, 24px inner padding (large) / 16px (compact).
- **Chips (word bank):** `rounded-2xl`, 8px/12px padding, pastel tint, 44px min height, indigo fill when selected/correct.
- **Progress bar:** slim rounded track `#E7E7FC`, indigo fill; numbered variant shows steps 1..N.
- **Shadows:** `shadow-soft` = `0 4px 16px rgba(48,48,60,0.08)`; `shadow-card` = `0 2px 8px rgba(48,48,60,0.06)`. No hard shadows.
- **Skeleton loaders:** pastel-gray rounded blocks matching card shapes; used for the loading state.
