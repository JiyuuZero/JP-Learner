<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported backup file -> IndexedDB | User-selected file (possibly malicious/oversized/corrupt) crosses into local storage, overwriting all progress. |
| IndexedDB persistence | Best-effort by default on iOS (7-day eviction) - a durability boundary, not a security one. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Malicious/corrupt backup overwrites progress | mitigate | importProgress validates shape + `backupVersion===1` BEFORE clearing; rejects unknown versions and does NOT clear on rejection; destructive-confirm dialog in the UI. Never eval; never dangerouslySetInnerHTML. |
| T-04-02 | Denial of service | Oversized/deeply-nested backup JSON | mitigate | Parse then shape-validate arrays before bulk put; reject non-conforming shapes early (fail-closed) rather than iterating unbounded structures. |
| T-04-03 | Tampering (data-loss) | IndexedDB evicted after 7 days (Core-Value guardrail) | mitigate | requestPersistence() on first interaction + export/import as the only backup, with a verified round-trip test shipping in this plan. |
| T-04-04 | Tampering | Botched schema migration drops progress | mitigate | Additive, idempotent `upgrade()` migrations keyed on oldVersion; backup is version-stamped so an old export can be re-imported; handle blocked/blocking. |
</threat_model>
