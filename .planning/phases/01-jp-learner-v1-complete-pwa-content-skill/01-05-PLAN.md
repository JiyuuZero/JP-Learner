renderTokens (verbatim, 01-RESEARCH.md Ruby Rendering):
  renderTokens(tokens: Token[], mode:'A'|'B'|'C', learned: Set<string>, romajiVisible: boolean): ReactNode[]
  - !isKanji token -> literal span
  - Mode C -> span(surface), no rt
  - Mode A -> if romajiVisible: <ruby>surface<rp>(</rp><rt>reading|romaji</rt><rp>)</rp></ruby> else span(surface)

  Mode A romaji granularity: apply romaji rt at WORD level (schema stores word-level romaji, not per-token) - A1 assumption.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| content JSON -> ruby/exercise DOM | Content strings (kanji/kana/romaji/es/tokens) render into React elements and `<ruby>`. |
| user typed input -> checker | User input is compared, never executed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | XSS via content rendered as HTML in `<ruby>`/cards | mitigate | Render content ONLY as React text children / `<ruby>` from structured token fields; NEVER dangerouslySetInnerHTML on any content field (grep gate in acceptance). React escapes by default. |
| T-05-02 | Tampering | Wrong furigana teaching wrong readings (Mode B linchpin) | mitigate | renderTokens reads authoritative readings from schema tokens only; NO in-app reading computation (grep gate forbids kuromoji/kuroshiro/wanakana); the 乗り込む/毎日 render tests prove correctness. |
| T-05-03 | Information disclosure | typed answers / progress leaked | accept | Single-user, local-only; input is compared in-memory, never transmitted (no network calls beyond same-origin content). |
</threat_model>
