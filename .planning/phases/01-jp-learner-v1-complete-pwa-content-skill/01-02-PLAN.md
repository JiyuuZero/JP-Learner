<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network/SW → app | Fetched content JSON and cached assets cross into the running app. |
| content JSON → DOM | Content strings render into React elements (text/`<ruby>` later). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | XSS via content JSON rendered as HTML | mitigate | Render content ONLY as React text children (auto-escaped). NEVER use `dangerouslySetInnerHTML` on any content field — enforced by grep gate in acceptance criteria. |
| T-02-02 | Tampering | Service-worker cache poisoning / stale cache serving broken app | mitigate | Use vite-plugin-pwa/Workbox (no hand-rolled SW); NetworkFirst for HTML+content JSON, cache-first only for hashed immutable assets; `registerType: 'autoUpdate'`. |
| T-02-03 | Denial of service | Content fetch fails → blank/blocked app | mitigate | ContentStore surfaces an explicit error state ("No se pudo cargar el contenido" + Reintentar); the shell still renders. |
| T-02-04 | Information disclosure | secrets/keys in the public bundle | mitigate | No secrets anywhere in the app (hard constraint); all AI lives in the local skill. The bundle makes zero network calls except same-origin content JSON. |
</threat_model>
