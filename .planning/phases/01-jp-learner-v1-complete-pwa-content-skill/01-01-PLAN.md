<!-- These are the FROZEN shapes. Copy verbatim; do not invent fields. Full spec + JSON is in 01-RESEARCH.md §Frozen Content Schema and §Sample Class JSON. -->

Generated TS types this plan produces (json-schema-to-typescript output — names come from $defs):
  ClassContent { schemaVersion: 1; classId: string; date: string; label: string; vocab?: Vocab[]; grammar?: Grammar[]; kanji?: Kanji[]; notes?: Note[]; }
  Token    { surface: string; reading: string; isKanji: boolean; kanji?: string[]; }
  Sentence { kanji: string; kana: string; romaji: string; es: string; tokens?: Token[]; }
  Vocab    { id: string; type: "vocab"; kanji: string; kana: string; romaji: string; es: string; pos?: string; tokens: Token[]; example?: Sentence; tags: string[]; }
  Grammar  { id: string; type: "grammar"; pattern: string; es: string; examples: Sentence[]; tags: string[]; }
  Kanji    { id: string; type: "kanji"; char: string; readings?: { on?: string[]; kun?: string[] }; es: string; seenIn: string[]; }
  Note     { id: string; type: "note"; es: string; tags: string[]; }

The 3 cross-field invariants (NOT expressible in JSON Schema — enforce imperatively in validate.mjs):
  1. tokens[].surface joined === item.kanji   (renderability: surface concat = written form)
  2. tokens[].reading joined === item.kana     (renderability: reading concat = full reading)
  3. every token with isKanji:true has a non-empty kanji[] array
ID rule: every item id matches ^<classId>:<type>:<slug>$ ; duplicate ids within a class are rejected.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Skill/author → content JSON | Hand-authored (and later skill-authored) JSON is untrusted structured data that the app will render. It crosses into the app as `<ruby>`/text. |
| content JSON → validate.mjs | The contract enforcement point: shape + cross-field invariants + ID determinism are checked here BEFORE any file is trusted/committed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | content class JSON (malformed/invalid shape) | mitigate | ajv compile+validate against the frozen draft-2020-12 schema with `additionalProperties:false` everywhere — unexpected fields fail closed (SKILL-03 / CONT-01). |
| T-01-02 | Tampering | tokens[] not matching kanji/kana (would produce wrong furigana → teaches wrong readings) | mitigate | The 3 imperative cross-field invariants in validate.mjs (surface-concat, reading-concat, per-kanji arrays) fail closed before write. |
| T-01-03 | Tampering | non-deterministic / colliding item IDs (would orphan SRS history — PROG-04) | mitigate | validate.mjs enforces `^<classId>:<type>:<slug>$` and rejects duplicate IDs within a class. |
| T-01-04 | Information disclosure | secrets accidentally embedded in content JSON | accept | Content is public-by-design (git-versioned class material); no secrets belong here. Reviewed at authoring; no keys in the app or content per hard constraint. |
</threat_model>
