<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Pages CDN -> installed PWA | The deployed bundle + SW are served to the client; stale/poisoned cache is the risk. |
| OS TTS voice -> app | The app requests speech from an OS-provided voice; wrong-voice output is a correctness risk, not a security one. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | Stale service worker serves old app / hides a new class after deploy | mitigate | vite-plugin-pwa `registerType:'autoUpdate'` + NetworkFirst for HTML+content JSON (Plan 02); live-URL check confirms a new class appears without a hard refresh. |
| T-06-02 | Information disclosure | secrets in the deployed public bundle | mitigate | No secrets anywhere (hard constraint); the app makes zero network calls except same-origin content JSON; deploy uses only public content. |
| T-06-03 | Elevation of privilege | over-broad GitHub Actions permissions | mitigate | Workflow grants only `pages: write` + `id-token: write` + `contents: read`; uses pinned official actions (deploy-pages/upload-pages-artifact). |
| T-06-04 | Tampering | TTS speaks Japanese with a non-Japanese voice (teaches wrong pronunciation) | mitigate | speakJa is a silent no-op when no ja-JP voice; the speaker button is hidden with no voice (never wrong-voice). |
</threat_model>
